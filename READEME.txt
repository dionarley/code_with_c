Para criar um stress test robusto em C++, o método mais eficiente é utilizar a Fórmula de Bailey–Borwein–Plouffe (BBP) ou a série de Chudnovsky. No entanto, para fins de aquecimento de CPU e teste de carga, uma implementação da Série de Leibniz ou de Monte Carlo com suporte a Multi-threading é mais direta e gera um consumo massivo de ciclos de processamento.Abaixo, preparei um código que utiliza std::thread para saturar todos os núcleos do seu processador, calculando o valor de $\pi$ através da integração numérica (método dos retângulos).

Como compilar e rodar
Para garantir que o teste realmente estresse a CPU, você deve compilar com otimizações de nível médio (-O2 ou -O3) e habilitar o suporte a threads:

Bash
g++ -O3 -pthread stress_pi.cpp -o stress_pi
./stress_pi

Por que isso estressa a CPU?FPU Intensity: O cálculo de $4.0 / (1.0 + x^2)$ é pesado em operações de ponto flutuante.Parallelism: O uso de std::thread::hardware_concurrency() garante que cada núcleo lógico (incluindo Hyper-threading) seja levado a 100% de uso.Atomic Ops: Embora a soma principal seja feita localmente em cada thread, a atualização final via compare_exchange_weak testa a consistência do cache L3 e da memória.Dica de Segurança: Se você estiver rodando isso em um sistema com refrigeração limitada (como alguns notebooks), monitore a temperatura. Esse código manterá a CPU em carga máxima até terminar o bilhão de passos. Quer uma versão que utilize instruções AVX/SIMD para um stress ainda mais pesado? 

Para calcular bilhões de dígitos de $\pi$, o método anterior de integração numérica não serve; ele é excelente para estresse térmico, mas péssimo para precisão (levaria eras para convergir para 1 bilhão de casas decimais).Para alta precisão em C++, a regra de ouro é usar a série de Chudnovsky — a mesma fórmula usada pelo Google e pela NASA para recordes mundiais. No entanto, para lidar com números de bilhões de dígitos, você precisará de uma biblioteca de precisão arbitrária como a GMP (GNU Multiple Precision Arithmetic Library).O Código do Stress Test de Alta PrecisãoAbaixo está uma implementação otimizada que utiliza a série de Chudnovsky.Nota: Este código requer a biblioteca libgmp. Em sistemas Arch/Void (que você utiliza), instale com sudo pacman -S gmp ou xbps-install -S gmp-devel.
